# Python Assignment – Part 3: File I/O, APIs & Exception Handling

## Overview

This project is part of my Python learning journey and focuses on understanding file handling, working with APIs, and handling errors using exception handling.

In this assignment, I built a simple program that:
- Reads and writes data into files
- Fetches data from an external API
- Handles runtime errors and logs them into a file

This helped me understand how real-world Python applications manage data and errors effectively.

---

## Objective

The main objective of this project is to:

- Learn how to read and write files in Python
- Understand how to fetch data using APIs
- Handle errors using try-except blocks
- Log errors into a file for debugging
- Build a simple and structured Python program

---

## Concepts Used

The following Python concepts are used in this project:

- File Handling (open, read, write, append)
- Exception Handling (try, except)
- Working with APIs using requests module
- Functions
- String formatting
- Date and Time module

## Explanation of Files

part3_api_files.py  
This is the main Python file that contains all the logic. It performs file operations, API calls, and handles errors.

python_notes.txt  
This file is created by the program and stores sample notes written using file handling.

error_log.txt  
This file is automatically generated when an error occurs. It stores error messages along with timestamps.

README.md  
This file explains the project, structure, and concepts used.

---

## How the Program Works

1. The program writes some content into a file (python_notes.txt).
2. It fetches data from an API using the requests library.
3. The data is processed and printed.
4. A deliberate error (division by zero) is created to demonstrate exception handling.
5. The error is caught and logged into error_log.txt with date and time.

---

## How to Run the Project

Step 1: Install required library

pip install requests

Step 2: Run the Python file

python part3_api_files.py

Step 3: Check output files

- python_notes.txt will contain written data
- error_log.txt will contain logged errors

The program prints product data fetched from the API and handles errors like:
